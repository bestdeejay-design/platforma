Спецификация сайта АО Платформа
Домен: aoplatforma.ru
Версия: 1.0
Дата: Февраль 2026

1. Общие положения
1.1. Назначение документа
Настоящая спецификация определяет функциональные, технические и дизайнерские требования к корпоративному сайту АО Платформа (aoplatforma.ru).
Сайт выполняет функции:
— Презентация компании и её продуктов (ЛОВИИ, Витрина, инфраструктурные решения)
— Информационный портал для партнёров, франчайзи и инвесторов
— Точка входа в экосистему платформенных сервисов
— Инструмент привлечения B2B-клиентов и регионального развития

1.2. Целевая аудитория
— Франчайзи и региональные партнёры
— Представители малого и микробизнеса
— Инвесторы и стратегические партнёры
— Представители государственных структур и фондов поддержки
— IT-специалисты и интеграторы
— СМИ и отраслевые аналитики

1.3. Ключевые цели сайта
— Формирование доверия к бренду АО Платформа
— Демонстрация технологического лидерства и экспертизы
— Конверсия посетителей в заявки на подключение / партнёрство
— Обеспечение прозрачной коммуникации с экосистемой
— Поддержка мультиязычного присутствия (RU / EN / CN)

2. Структура сайта
2.1. Главная страница
— Hero-блок: ключевое УТП, визуал с анимацией, CTA-кнопки
— Блок «О компании»: миссия, ценности, цифры
— Продукты и решения: карточки ЛОВИИ, Витрина, инфраструктура
— Для кого: сегментация по аудиториям (франчайзи, бизнес, инвесторы)
— Партнёры и интеграции: логотипы, кейсы
— Новости и события: анонсы, публикации, медиа
— Footer: контакты, соцсети, документы, навигация

2.2. Раздел «Продукты»
2.2.1. ЛОВИИ
— Описание концепции: оператор лояльности, баллы, СБП
— Сценарии использования: для клиентов, бизнеса, корпораций
— Техническая архитектура: модули, интеграции, безопасность
— Документация: ссылки на API, спецификации, FAQ

2.2.2. Витрина
— Цифровая витрина для малого бизнеса
— Функционал: каталог, остатки, заказы, аналитика
— Интеграции: 1С, кассы, логистика, БПА
— Кейсы и результаты: метрики, отзывы

2.2.3. Инфраструктурные решения
— КИП (Корпоративная информационная платформа)
— SaaS-модель для франчайзи
— API и white-label возможности

2.3. Раздел «Партнёрам»
— Франшиза City Franchise: условия, потенциал, калькулятор
— Для рекламодателей: возможности Мобиап, форматы, тарифы
— Для интеграторов: API-документация, sandbox, поддержка
— Для инвесторов: презентация, финансовая модель, контакты

2.4. Раздел «Компания»
— О АО Платформа: история, миссия, команда
— Юридическая структура: АО Точка, АО Аксиома, ООО Атмосфера, ПК Академия
— Документы: лицензии, аккредитации, политики
— Карьера: вакансии, ценности, процесс найма

2.5. Раздел «Ресурсы»
— Блог: аналитика, кейсы, тренды цифровой экономики
— Документация: спецификации, API, руководства
— Медиа: пресс-кит, логотипы, брендбук
— Поддержка: FAQ, тикеты, контакты

2.6. Служебные страницы
— Контакты: форма обратной связи, реквизиты, карта
— Политика конфиденциальности и обработки ПДн
— Пользовательское соглашение
— Страница 404 с навигацией

3. Дизайн и UI/UX
3.1. Визуальная концепция
— Стиль: современный, технологичный, с элементами glassmorphism
— Цветовая палитра:
• Основные: тёплые землистые тона (терракота #C17A5C, шалфей #8A9A8B, охра #D4A76A)
• Фоновые: мягкие градиенты, текстурированные поверхности
• Акценты: мягкие цветочные и природные оттенки
— Типографика: чистые гротески с хорошей читаемостью, поддержка кириллицы

3.2. Темы оформления
— Тёмная (default): глубокие тона, мягкие свечения, акценты на контенте
— Светлая высокая контрастность: оптимизирована для чтения, доступность AA/AAA
— Яркая: современная палитра с акцентами на интерактивных элементах
— Переключение тем: через настройки в хедере, сохранение выбора в localStorage

3.3. Адаптивность
— Mobile-first подход
— Поддержка разрешений: 320px – 4K
— Адаптивная сетка, гибкие изображения, touch-оптимизация
— Корректное отображение на планшетах и десктопах

3.4. Интерактивные элементы
— Микроанимации при наведении и взаимодействии
— Плавные переходы между состояниями
— SVG-иконки из единого спрайта для визуальной консистентности
— Интерактивные карточки продуктов с hover-эффектами

4. Функциональные требования
4.1. Мультиязычность
— Поддержка языков: русский (default), английский, китайский
— Переключение языка в хедере, сохранение выбора
— Автоматическое определение языка браузера (с возможностью изменения)
— Отдельные URL-структуры: /ru/, /en/, /cn/

4.2. Навигация
— Десктоп: горизонтальное меню в хедере + дублирование в футере
— Мобильный: фиксированный нижний бар с 5 основными кнопками
— Хлебные крошки для глубоких разделов
— Поиск по сайту с автодополнением

4.3. Формы и взаимодействие
— Валидация полей в реальном времени
— Защита от спама (reCAPTCHA v3 или аналог)
— Отправка заявок через API с подтверждением по email/SMS
— Интеграция с CRM для обработки лидов

4.4. Контент-менеджмент
— Админ-панель для управления контентом (тексты, медиа, документы)
— Версионирование публикаций
— Предпросмотр изменений перед публикацией
— Ролевая модель доступа для редакторов

4.5. Интеграции
— CRM-система для обработки заявок
— Аналитика: Яндекс.Метрика, Google Analytics 4
— Чат-поддержка: виджет с маршрутизацией
— Авторизация через VK ID, Telegram, Mail.ru (опционально)

5. Технические требования
5.1. Технологический стек
— Frontend: React / Next.js с SSR для SEO
— Backend: Node.js / Python (API-шлюз)
— CMS: Headless CMS (Strapi / Directus) или кастомное решение
— Хостинг: облачная инфраструктура с гео-репликацией в РФ

5.2. Производительность
— Время загрузки первой краски (FCP): ≤ 1.2 сек
— Время интерактивности (TTI): ≤ 2.5 сек
— Lighthouse Performance: ≥ 90
— Оптимизация изображений: WebP, lazy loading, CDN

5.3. Безопасность
— HTTPS (TLS 1.3) на всех страницах
— Защита от XSS, CSRF, SQL-инъекций
— Rate limiting для API-эндпоинтов
— Регулярное сканирование уязвимостей

5.4. SEO и доступность
— Семантическая разметка (Schema.org)
— ЧПУ, мета-теги, Open Graph, Twitter Cards
— Соответствие WCAG 2.1 AA
— Sitemap.xml, robots.txt, canonical-теги

5.5. Мониторинг и аналитика
— Сбор метрик производительности (RUM)
— Логирование ошибок (Sentry / аналог)
— Отслеживание конверсий и пользовательских сценариев
— Дашборды для маркетинга и продукта

6. Контент-стратегия
6.1. Тон коммуникации
— Профессиональный, но человечный
— Акцент на выгодах для партнёров и бизнеса
— Избегание излишнего технического жаргона в публичных разделах

6.2. Типы контента
— Тексты: лаконичные, структурированные, с подзаголовками
— Визуалы: фотореалистичные иллюстрации с мягкими тенями
— Видео: короткие презентации продуктов (до 90 сек)
— Документы: PDF-спецификации, презентации, отчёты

6.3. Обновление контента
— Блог: минимум 2 публикации в месяц
— Новости: по мере значимых событий
— Документация: синхронизация с актуальными версиями продуктов

7. Этапы реализации
7.1. Этап 1: MVP (6–8 недель)
— Главная страница + разделы «Продукты», «Контакты»
— Базовая навигация, формы заявок, аналитика
— Одна тема оформления (тёмная), русский язык

7.2. Этап 2: Расширение (4–6 недель)
— Полная структура разделов
— Мультиязычность (EN), переключение тем
— Интеграции: CRM, чат, авторизация

7.3. Этап 3: Оптимизация (2–4 недели)
— A/B-тестирование CTA и контента
— Улучшение производительности и SEO
— Поддержка китайского языка, расширенная аналитика

8. Метрики успеха
— Конверсия посетителей в заявки: ≥ 3%
— Среднее время на сайте: ≥ 2:30 мин
— Показатель отказов: ≤ 40%
— Lighthouse Accessibility: ≥ 95
— Удовлетворённость пользователей (опрос): ≥ 4.5/5
